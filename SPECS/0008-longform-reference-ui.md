# 0008 Longform Reference UI

## 目标

为长文参考看板定义一个独立界面。

这个界面不影响现有的：

- Aesthetic Board
- Creator Reactor

它是第三个独立空间，服务“长文样本收藏 + AI 写法拆解”。

## 第一性原理

长文参考板不是“文件管理器”。

创作者进入这里，不是为了：

- 找文件
- 看文件夹层级
- 做后台管理

而是为了：

- 浏览值得学习的长文
- 被标题与气质吸引点开
- 阅读原文
- 理解为什么它写得好

所以它的界面不能像：

- 常规列表页
- 文档后台
- 文件夹网盘

它应该像：

- 一本可翻阅的编辑参考集
- 一个策展式样本书架
- 一本“长文写法年鉴”

## 设计方向选择

在两个参考方向里，我建议走：

### 主方向：日杂 / 编辑部 / 杂志排版感

原因：

- 更适合长文的阅读气质
- 更容易承载标题、作者、出处、AI 拆解这些文本信息
- 不会像普通书架那样太重、太拟物
- 能和当前产品的纸张感、留白感自然衔接

### 辅助语言：卡片书架 / 翻阅感

原因：

- 可以借“层叠卡片 / 册页”表达收藏与翻阅
- 适合做精选样本的浏览动势

### 不建议做成纯文件夹

因为：

- 太像知识管理工具
- 容易把“样本学习”做成“资料归档”
- 和产品现在的审美方向冲突

## 页面结构

## 1. Shelf

这是入口页。

不是列表，不是表格，而是一张“编辑参考墙”。

### 组成

- 左侧或中央：精选主样本大卡
- 周围：次级样本卡 / 书脊卡 / 摘要卡
- 顶部：很轻的标题区
- 底部：少量筛选或切换

### 视觉感受

- 低饱和纸色背景
- 杂志级留白
- 大字标题 + 小字脚注
- 卡片有轻微层叠与翻阅感
- 不做沉重拟物书柜

## 2. Entry Detail

点击一篇样本后，进入一篇独立的阅读详情页。

这页不是弹窗，而像一本展开的样本册。

### 布局建议

- 左：原文阅读区
- 右：AI 拆解区

或

- 上：标题与出处
- 中：原文
- 右侧边栏：AI 拆解

原则是：

- 原文始终是主角
- AI 是旁批，不是盖过原文的解释层

## 3. Compare Layer

后续可选。

允许并排看两篇长文样本，比较：

- 标题策略
- 开头方式
- 结构差异

首版先不做，但视觉系统要为此留余地。

## 视觉语言

## 1. 整体气质

- 日杂 / 编辑参考册
- 低饱和米白、灰褐、墨色
- 像样本书，不像工具台

## 2. 排版

- 大标题可以更有杂志感
- 副信息用很轻的衬线或窄体小字
- 卡片上少量竖排混排可以出现，但要克制
- 标题、作者、出处三层信息必须清楚

## 3. 卡片

卡片建议分三类：

### Hero Card

- 放精选样本
- 大标题
- 一句“为什么值得学”

### Reference Card

- 放普通样本
- 标题 + 出处 + 短备注

### Spine Card

- 像书脊或索引卡
- 只露标题关键词 / 编号 / 类型
- 服务浏览节奏

## 4. 装饰与物件

可以有很轻的：

- 书页边线
- 纸张压痕
- 浮雕编号
- 印刷式小号注记

不做：

- 厚重书架木纹
- 大面积拟物阴影
- 假文件夹标签

## 信息密度原则

入口页只回答：

- 这篇是什么
- 为什么值得点开
- 这类样本整体在学什么

不要在入口页就显示整套 AI 分析。

详情页才展开：

- 为什么好
- 框架
- 技巧
- 共鸣
- 爆款潜力

## AI 在界面中的位置

AI 不是首页主角。

AI 是进入详情页后的一层“旁批”。

### 视觉表达

- 更轻
- 更像编辑批注
- 更像 margin note / analysis panel

不要做：

- 大面积 AI 结果块
- 聊天窗口
- 自动飞出来的解释面板

## 与现有界面的关系

## 独立入口

建议以后作为单独入口存在：

- 不是塞进 Reactor
- 不是挂在 Aesthetic Board 的某个角落

可以是：

- 第三个板块
- 或左侧工具入口中的独立 view

但打开后应该进入完整新界面，而不是 modal。

## 共享的东西

- 品牌色系
- 纸张感和光感
- 留白逻辑
- 按钮语言的克制感

## 不共享的东西

- 周历结构
- 情绪板式图片陈列
- Reactor 的自由画布

## 首版实现建议

先不要上复杂 3D 书架。

首版做：

1. 独立入口
2. 杂志式 shelf 首页
3. 样本详情页
4. AI 拆解侧栏

把“像一本样本参考册”先做成立。

## 验收信号

- 用户第一眼知道这不是文件夹，而是可翻阅的参考库
- 首页会让人想点开某篇样本继续看
- 详情页里，原文和 AI 拆解能自然并存
- 整体气质比常规知识库更高级、更耐看
